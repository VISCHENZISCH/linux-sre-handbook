Questions d'entretien DevOps/SRE

Préparez vos recrutements avec les questions techniques les plus posées en entretien d'ingénierie.

Quelle est la différence entre Load Average et utilisation CPU ?
Le Load Average mesure le nombre de processus en attente d'exécution ou bloqués par des E/S (disques ou réseau) dans la file d'attente du noyau (R + D state). L'utilisation CPU, quant à elle, représente la proportion de temps pendant laquelle le processeur exécute du code actif. Un système peut avoir un Load Average très élevé (ex: 20 sur un CPU à 4 cœurs) avec une utilisation CPU à 10% si le système est bloqué par des accès disques saturés.
Qu'est-ce qu'un Zombie Process et comment s'en débarrasser ?
Un processus zombie est un processus dont l'exécution est terminée, mais qui possède toujours une entrée dans la table des processus du noyau car son processus parent n'a pas encore lu son statut de sortie (via l'appel système wait() ou waitpid()). Les zombies ne consomment pas de ressources (CPU ou RAM), seulement un PID. Pour s'en débarrasser, on doit envoyer le signal `SIGCHLD` au parent, le tuer (le zombie sera adopté par `init`/PID 1 qui le nettoiera), ou redémarrer le service parent.
Expliquez la différence fondamentale entre GitOps Push-based et Pull-based.
Dans le modèle Push-based (ex: GitLab CI), un orchestrateur externe pousse directement les modifications sur le cluster (via kubectl apply) lors d'un commit. Dans le modèle Pull-based (ex: ArgoCD, Flux), un agent interne au cluster surveille le dépôt Git et réconcilie l'état du cluster si un écart est détecté. Le modèle Pull-based est plus sécurisé car il n'expose aucune clé d'accès admin du cluster vers l'extérieur.

Feuille de route d'apprentissage (Roadmap)

Suivez cette progression logique et chronologique pour monter en compétences sur la stack moderne.

Étape 1 : Fondations Linux & Shell 100h
Maîtriser les commandes d'administration basiques, la gestion des permissions, l'architecture du système de fichiers, l'écriture de scripts Bash avancés et la gestion de processus.
Étape 2 : Réseau avancé & Sécurité 80h
Comprendre en profondeur la pile TCP/IP, configurer des firewalls UFW/nftables, durcir le protocole SSH, mettre en place des certificats SSL/TLS et gérer les serveurs web (Nginx).
Étape 3 : Conteneurisation & CI/CD 120h
Maîtriser Docker, écrire des Dockerfiles optimisés en multi-stage, concevoir des environnements multi-conteneurs avec Docker Compose et automatiser les builds dans des pipelines GitHub Actions.
Étape 4 : Orchestration (Kubernetes) 150h
Apprendre à déployer et administrer des applications sur Kubernetes, comprendre l'architecture interne (etcd, scheduler), configurer des volumes persistants, des Ingress et utiliser ArgoCD.
Étape 5 : SRE, Observabilité & Cloud 100h
Mettre en œuvre la stack Prometheus / Grafana / Loki, comprendre la théorie SRE (budgets d'erreurs, SLOs) et maîtriser un provider Cloud majeur (AWS/GCP) avec Terraform.
