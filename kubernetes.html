Kubernetes Orchestration

Gestion de conteneurs à grande échelle sur des clusters de serveurs distribués.

~15 min de lecture Expert / Orchestrateur Mis à jour Mai 2026

Commandes kubectl Critiques

Pilotez votre cluster et diagnostiquez les pannes avec les commandes de base `kubectl`.

Analyse des pods

# kubectl get pods -A
# kubectl describe pod mypod
# kubectl logs mypod -f

Interactions

# kubectl exec -it mypod -- sh
# kubectl port-forward mypod 8080:80
# kubectl apply -f deploy.yml

Manifestes YAML Prêts pour la Production

Un déploiement résistant intégrant des probes de vie et des contraintes de ressources.

# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-app
  labels:
    app: web
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web
  template:
    metadata:
      labels:
        app: web
    spec:
      containers:
      - name: nginx
        image: nginx:1.21-alpine
        resources:
          limits:
            cpu: "500m"
            memory: "256Mi"
          requests:
            cpu: "100m"
            memory: "128Mi"
        ports:
        - containerPort: 80
        livenessProbe:
          httpGet:
            path: /
            port: 80
          initialDelaySeconds: 15
          periodSeconds: 20

Kubernetes vs Docker

Comprendre la différence fondamentale d'échelle entre ces deux outils indispensables.

Critère Docker Kubernetes (K8s)
Échelle Mono-hôte (un seul serveur) Multi-hôtes (Cluster de serveurs)
Résilience Restart manuel ou basique Auto-healing, réplicas distribués
Réseau Ponts isolés internes simples Réseau plat distribué (CNI: Calico, Flannel)
Mises à jour Interruption temporaire de service Rolling updates sans temps d'arrêt
